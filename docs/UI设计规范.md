## 一、设计原则

1. **柔和极简**：圆润形状、柔和阴影、大量留白，比纯 flat 多一层质感
2. **内容优先**：UI 服务于学习内容，不喧宾夺主
3. **减少决策疲劳**：核心操作（记录、复习）始终一步可达
4. **响应式优先**：同一套设计语言适配桌面和手机

---

## 二、设计趋势依据（2025-2026）

### 采纳的趋势

| 趋势 | 应用场景 | 说明 |
|------|---------|------|
| 柔和极简 | 全局 | 圆角卡片（12-16px）、柔和阴影、暖色底色、大量留白 |
| Bento Grid | 首页仪表盘 | 模块化卡片拼合，不同尺寸展示不同信息密度 |
| 精制玻璃拟态 | 导航栏、弹窗 | 半透明毛玻璃效果仅作局部点缀，不滥用 |
| 暖色基底 | 全局底色 | 2026 趋势从冷灰转向暖白/米色 |
| 微交互动效 | 翻卡、按钮、进度条 | 200-500ms 细节动画，增强操作反馈 |

### 明确不采用的

| 过时趋势 | 原因 |
|---------|------|
| Neumorphism | 2020 年流行，现已过时，且有无障碍问题 |
| 彩虹/强烈渐变 | 已被柔和渐变取代 |
| 纯 Flat 无层次 | 缺乏深度信号，影响信息层级识别 |
| 弹窗轰炸 | 学习场景中打断流程体验极差 |

### 参考产品

| 产品 | 参考点 |
|------|--------|
| Duolingo | 学习类 App 的圆润风格、gamification 元素、鼓励性文案 |
| Notion | Bento 布局、干净的信息层级 |
| Linear | 仪表盘动效、暗色模式处理 |
| Quizlet | 卡片组展示方式、创建入口设计 |

## 四、通用 UI 规范（三套方案共用）

### 字体

- 主字体：`Inter`（已在 prototype 中确定）
- 备选：`Plus Jakarta Sans`、`DM Sans`
- 字号层级：
  - 页面标题：24-32px / font-weight 700
  - 卡片标题：18-20px / font-weight 600
  - 正文：16px / font-weight 400
  - 辅助文字：14px / font-weight 400
  - 标签/徽章：12px / font-weight 500

### 响应式导航

| 屏幕 | 布局 |
|------|------|
| 桌面端（≥768px） | 顶部导航栏，Tab 链接水平排列，右侧用户头像 |
| 手机端（<768px） | 底部导航栏，4 个 Tab 图标 + 文字标签 |

### 桌面端头像交互（v1 极简）

- 头像位置：顶部导航右侧，圆形按钮（32px）
- 点击头像展开下拉菜单，包含两项：
  - 账号信息（即将上线）
  - 退出登录
- 关闭方式：再次点击头像、点击菜单外区域、按 `Esc`
- 视觉风格：下拉面板圆角 12px，阴影 `0 8px 24px rgba(0,0,0,0.12)`，与全局弹窗层级一致

### 四个主 Tab 布局一致性约束（v1 必须）

- 四个主 Tab 页面（首页/记录/复习/卡片组）统一使用同一外层宽度：
  - `.topnav-inner`：`max-width: 1100px`
  - `.main`：`max-width: 1100px`
- 桌面端统一横向内边距：导航与内容容器均为 `padding-inline: 24px`；手机端均为 `16px`
- 记录页若需要更窄阅读宽度，必须在 `.main` 内新增内层容器（如 `.content-narrow`）控制，不允许改单页 `.main` 宽度
- 为避免不同页面因滚动条出现/消失导致横向跳动，全局启用稳定滚动条策略（二选一）：
  - `html { scrollbar-gutter: stable; }`（优先）
  - 或 `body { overflow-y: scroll; }`（兼容兜底）
- 以上规则同样适用于 React 版本，禁止按页面各自定义外层宽度

### 间距系统

- 基础单位：4px
- 常用间距：8px / 12px / 16px / 24px / 32px / 48px
- 页面内边距：桌面 32-48px，手机 16-20px
- 卡片内边距：20-24px
- 卡片间距：16px

### 圆角

- 卡片 / 大容器：16px
- 按钮 / 输入框：12px
- 标签 / 徽章：8px（或 full rounded）
- 头像：full rounded

### 阴影

- 卡片默认：`0 1px 3px rgba(0,0,0,0.04), 0 1px 2px rgba(0,0,0,0.06)`
- 卡片悬浮：`0 4px 12px rgba(0,0,0,0.08)`
- 弹窗 / 模态框：`0 8px 24px rgba(0,0,0,0.12)`

### 动效

| 交互 | 时长 | 缓动函数 |
|------|------|---------|
| 按钮悬浮/按压 | 150ms | ease-out |
| 卡片翻转（复习） | 300-400ms | ease-in-out |
| 页面切换 | 200-300ms | ease-out |
| 弹窗出现 | 250ms | ease-out |
| 进度条填充 | 500ms | ease-out |

### 卡片组配色（Anki 式，三套方案通用）

| 卡片状态 | 颜色用途 | 建议色值 |
|---------|---------|---------|
| 新卡 | 蓝色标记 | `#3B82F6` |
| 学习中 | 红色/橙色标记 | `#EF4444` |
| 待复习 | 绿色标记 | `#22C55E` |

### Anki 四档按钮配色

| 档位 | 颜色 | 说明 |
|------|------|------|
| Again | 红色 `#EF4444` | 需要重学 |
| Hard | 橙色 `#F97316` | 有明显问题 |
| Good | 绿色 `#22C55E` | 基本正确 |
| Easy | 蓝色 `#3B82F6` | 完全掌握 |

## 五、页面清单（Mock 阶段）

### 第一步：HTML 静态视觉稿（风格评审）

每套方案各做 2 个页面：

| 页面 | 核心内容 |
|------|---------|
| 首页 | 问候语 + 今日待复习数 + 最近卡片组 + 记录入口 |
| 记录页 | 输入框 + 生成英文按钮 + AI 结果展示 + 卡片组选择弹窗 |

### 第二步：React 完整交互原型（选定风格后）

| 页面 | 核心交互 |
|------|---------|
| 首页 | 跳转复习、跳转记录、查看卡片组 |
| 记录页 | 输入 → 模拟生成英文 → 展示结果 → 选择组 → 保存 |
| 复习页 | Deck 列表 → 进入 Deck → 翻卡流程 → 输入英文 → 模拟评分 → 选择档位 → 下一张 → 总结页 |
| 卡片组页 | 查看所有组 → 进入组 → 查看卡片列表 → 编辑/删除/移动卡片 |
| 登录/注册 | 表单交互（mock，不连后端） |

### 空状态页面

| 页面 | 空状态内容 |
|------|-----------|
| 首页 | "还没有学习卡片"，主按钮"创建第一张卡片" |
| 复习页 | "暂无待复习内容"，主按钮"去记录新内容" |
| 卡片组页 | 始终显示默认组（0 张）与"创建卡片组"入口 |
